## Administration

With an authenticated client, you can control the server.

### Establishing a connection

First, a client must authenticate itself
with the corresponding password from `server.properties`:

```xml
<protocol><authenticate password="examplepassword"/>
```

An authenticated admin client is notified
when a :t[computer player]{#player} connects via a `JoinRoomRequest`:

```xml
<joinedGameRoom roomId=ROOMID playerCount=X />
```

- **roomId**: a character string generated by the server that identifies the match
- **playerCount**: the number of players in the room after the new player has joined

### Actions in a room

An admin client can register as an observer of a room,
thereby receiving all messages concerning the game:

```xml
<observe roomId=ROOMID />
```

An admin client can pause or resume a game:

```xml
<pause roomId=ROOMID pause="true/false" />
```

An admin client can request the next move in a paused game:

```xml
<step roomId=ROOMID />
```

An admin client can cancel a game:

```xml
<cancel roomId=ROOMID />
```

An admin client can prepare a game room:

```xml
<prepare gameType="pluginId" pause="true">
  <slot displayName="p1" canTimeout="true" reserved="true"/>
  <slot displayName="p2" canTimeout="true" reserved="true"/>
</prepare>
```

- **pluginId**: identifies the game (for Hey, Thanks for the Fish!: `swc_2023_pengins`)
- **pause**: whether the created game starts paused after all players have joined

The server may respond to this with reservation codes for the players to join.

### Server settings

The server can be configured in the `server.properties` file,
which is downloaded together with the server.
The following values can be configured in it:

- **password**: local administrator password (default is `examplepassword`)
- **paused**: whether an automatically created game should be initially paused or not (default is paused)
